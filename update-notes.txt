April 20, 2026  [Boilerplate 3.1 — Lead Architect Audit]
[ARC-001] CRITICAL: Fix error message leakage in GlobalErrorHandler:
     :: Non-Fiber errors previously passed raw err.Error() to clients —
        could expose SQL queries, internal paths, or stack traces
     :: GlobalErrorHandler now returns generic "An unexpected error occurred."
     :: Helper functions (InternalServerError, BadRequestError, GatewayTimeoutError)
        changed from handler-factory pattern func(error) fiber.Handler to direct
        func(fiber.Ctx, string) error — callers explicitly provide safe messages
     :: Removed per-status-code switch cases that duplicated the default branch
[ARC-002] MAJOR: Add 15-second startup timeout to database initialization:
     :: initSingle now uses context.WithTimeout(context.Background(), 15s)
     :: Process fails fast if PostgreSQL is unreachable instead of hanging forever
[ARC-003] MINOR: Document worker pool shutdown context behaviour:
     :: cancel() -> close(jobs) -> wg.Wait() order is intentional for cooperative shutdown
     :: Jobs needing to survive shutdown should create their own internal context
[ARC-004] MAJOR: Add database connection close on graceful shutdown:
     :: New Registry.Close() method closes all registered *postgres.Database connections
     :: Called in main.go after pool.Stop() during graceful shutdown
     :: Prevents pgx pool leaks and lingering idle connections on PostgreSQL
[ARC-005] MAJOR: Fix multitenant_test to exercise actual middleware:
